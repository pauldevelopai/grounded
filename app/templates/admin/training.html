Strategy Generation → Output

How user profile and activity data is used to generate personalized AI strategies.

How Profile & Activity Influences Strategy Output

Profile → RAG Search
  • • Use Cases → targeted tool search
  • • Risk Level → security content
  • • Data Sensitivity → privacy content
  • • Deployment → cloud/sovereign docs
  • • Org Type → org-specific content
If empty: generic search only
Profile → LLM Prompt
  • • Role (default: journalist)
  • • Org Type (default: newsroom)
  • • Risk Level (default: moderate)
  • • Data Sensitivity (default: standard)
  • • Budget (default: limited)
Defaults used if profile empty
Activity Data (Optional)
  • • Tool searches → added to prompt
  • • Tool Finder needs → use case hints
  • • Browsed sections → topic interest
User must check "Include activity"
RAG → LLM Context
  • • Search queries from profile
  • • Top 10 chunks retrieved
  • • Chunks become LLM context
  • • Citations tracked
Better profile = better retrieval
